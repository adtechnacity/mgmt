# 03 — Terminology

## Maven object glossary

| Term | Definition |
|------|------------|
| **Account** | Top-level client or partner. Examples: CapitalOne Shopping, Fisher Investments, Smarter Advisors. |
| **O&O page** | Owned and Operated page. The Instapage or Plasmic site the LP lives on. Examples: CCF (couponcodefinder.com), OST (onlineshoppingtools.com), GFC (getfreecoupons.com). Replaces "Workspace" from Asana terminology. |
| **Initiative** | Configuration unit that ties together account, O&O page, traffic source, and device. Every LP lives inside an initiative. Created once per platform/partner/device combination. |
| **Initiative group** | The organizational grouping above initiatives. Example: "COS US Desktop", "COS US Mobile". Determines how initiatives appear in Reports drill-downs. |
| **Data Source (DS)** | Maven's numeric identifier for a traffic source. Embedded in every Media URL. Examples: DS12 = Facebook, DS18 = Taboola, DS14 = Bing. |
| **Landing Page (LP)** | A Maven tracking record tied to one physical Instapage/Plasmic page. Contains pre-lander URL, client landing URL, LP name, EC/popup/continuation settings, and experiment flag. |
| **Landing Page Group (LPG)** | Container for one or more LPs. The LPG ID goes into the live campaign URL. Enables traffic distribution and A/B testing between LP variants. Every LP must be in an LPG. |
| **Pre-lander URL** | The Instapage/Plasmic page URL with all tracking parameters appended. This is the field Sheba fills in Maven for front door (pre-lander on) campaigns. Becomes the first page the user sees after clicking the ad. |
| **Client landing URL** | The final destination URL — the COS browser extension page, Fisher page, etc. The user is redirected here after the pre-lander (front door) or directly (back door). |
| **Media URL** | The URL that goes into the ad platform. Generated by Maven. For front door: uses alias domain (`cos-rd.com/DS/LPG` + params). For back door: uses the Instapage URL directly + params. |
| **QA URL** | Media URL + `&atnqa=1`. Used to test the redirect chain without generating live tracking data. Always provided alongside the Media URL at handoff. |
| **Alias domain** | The redirect domain used in front door Media URLs. Format: `cos-rd.com/[DS]/[LPG ID]`. Bing and Facebook/Google do not support alias domains. |
| **Pre-lander on** | Tracking mode where the user lands on the O&O page first, then is redirected to the client destination. Used for Taboola, Outbrain, MSN, Bing, most native platforms. Formerly called "front door." |
| **Pre-lander off** | Tracking mode where the user goes directly to the client/O&O destination. No pre-lander page. Used for Facebook and Google. Formerly called "back door." |
| **UTM source** | A key-value block in Google Tag Manager that identifies which holdup/welcome screen experience to show. Controls which experience the user sees after clicking the ad. Required for all campaigns. Takes 5–15 minutes to propagate after creation. |
| **EC** | Exit Capture. OptinMonster popup that appears when the user moves to leave the page. Controlled by `&ec=0` (off) or `&ec=1` (on) in the pre-lander URL. |
| **Popup** | The HoldUp experience — a window that appears when the user clicks the CTA. Controlled by `&popup=1` (on) in the URL. |
| **Continuation** | A multi-step user flow attached at the LP level. Defines where the user goes after clicking the CTA: pre-lander → intermediate page → final destination. Exception to the LPG rule — continuations are not used with LPGs. |
| **ExpID** | The GrowthBook experiment identifier. Appended to LP URLs as `&expid=[value]` after QA. Required for experiment tracking in GrowthBook. Added after QA to keep test data clean. |
| **MavenTrack** | The redirect management and traffic tracking system. Handles all URL redirects and fires attribution (ATN ID) on ad click. |
| **ATN ID** | The unique click identifier. Populated by the `{atnid}` macro in the pre-lander URL. Used for attribution tracking. |
| **Angel Rojas** | The person who publishes the GTM container after Sheba creates a new UTM source. Required step before the new UTM source goes live. |

## Asana form field → Maven field mapping

### Direct auto-mapping (value copies unchanged)

| Asana field | Maven field | Notes |
|-------------|-------------|-------|
| Instapage URL | Pre-lander URL (base) | Front door: becomes pre-lander URL + params. Back door: becomes landing URL + params. |
| Experiment Page (yes/no) | Is Experiment toggle | yes → ON, no/null → OFF |
| Extra Steps/Multi Offer | Pre-lander + Continuation config | Signals that continuation setup is needed |

### Translation logic required (system must know the rules)

| Asana field | Maven field | Translation rule |
|-------------|-------------|-----------------|
| Platform | Data Source + tracking type + URL format | Facebook → DS12, pre-lander off, no alias, `{{macros}}`. Taboola → DS18, pre-lander on, alias, `{macros}`. Bing → DS14, pre-lander on, no alias (adtechnacity.com URL), BTN format. Outbrain → DS21, pre-lander on, alias, `{{macros}}`. |
| Revenue Stream | Account → Initiative (account level) | COS → CapitalOne Shopping account. Fisher → Fisher Investments account. |
| Workspace (O&O page) | Initiative (narrow selection) | CCF → CouponCodeFinder initiative. OST → OnlineShoppingTools initiative. |
| Desktop/Mobile | Initiative Group | Desktop → COS US Desktop group. iOS Mobile → COS US Mobile group (CTC naming). |
| Angle(s) | LP name (partial) | Angle becomes part of LP naming convention. e.g. "Senior Travel" → "FB - Senior Travel Angle - Coupons CTA - EC=0" |
| utm_source | GTM config (outside Maven) | "new source required" → Sheba goes to GTM, creates key-value block, attaches LPG ID. "use existing" → select from existing UTM sources. |

### Asana-only fields (no Maven equivalent)

| Asana field | Notes |
|-------------|-------|
| ATN Priority | Routing only — determines Sheba's work order |
| LP Build Needed | Determines whether Bao builds the page first |
| Shared ID required | Analyst sets this in the ad platform (C2C/CTC) |
| Performance Source | Set at initiative level, not per-LP |

### Sheba writes back to Asana (output fields)

| Maven output | Asana field |
|--------------|-------------|
| LP ID + LPG ID | Maven ID field (e.g. "LP 16215 / LPG 16216") |
| Generated Media URL | Maven URLs field |

## Naming conventions

**LP name format:**
`[Platform abbrev] - [Angle] Angle - [CTA type] - EC=[value]`

Example: `FB - Senior Travel Angle - Coupons CTA - EC=0`

**UTM source naming format:**
`[brand][platform]` — lowercase, no spaces

Example: `ccf_tbfa` (CCF + Travel/Hotel Facebook)

**Initiative ID format:**
`[Partner code][sequential number]`

Example: `COS115` = CapitalOne Shopping's 115th initiative

**LPG naming convention:**
`Master [Angle] LPG`

Example: `Master Pricing Angle LPG`
