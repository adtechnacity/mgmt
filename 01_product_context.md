# 01 — Product context

## What Maven is

Maven is an internal ad operations platform built by Adtechnacity. It has two primary surfaces:

- **Maven Manage** — where ad ops sets up and configures campaigns. Covers accounts, initiatives, landing pages, landing page groups, data sources, and URL generation.
- **Maven Reports** — where analysts monitor performance. Covers redirects, visits, conversions, CPA, RPA, and experiment data by initiative, LP, and LPG.

The two surfaces are currently disconnected — there are no cross-links between Manage and Reports, and they use separate navigation structures.

## What the Build feature is

The **Build** feature is a new entry point inside Maven Manage that we are designing. It allows ad ops specialists to create a new campaign — specifically a Landing Page (LP) and Landing Page Group (LPG) — from a single guided form, without having to navigate through multiple separate screens.

The Build feature is accessed via the main Maven screen queue (from an Asana ticket) or via a "+ Create" button.

Current status: wireframes in progress. Not yet built. Claude Code is being used to implement the interactive wireframes.

## Key objects in Maven

**Account** — the top-level client or partner. Examples: CapitalOne Shopping, Fisher Investments, Smarter Advisors. All work is organized under an account.

**O&O page** — "Owned and Operated" page. The Instapage or Plasmic website the landing page lives on. Examples: CCF (Coupon Code Finder), OST (OnlineShoppingTools), GFC (GetFreeCoupons). Each O&O page corresponds to a domain.

**Initiative** — the configuration unit that connects an account, O&O page, traffic source, and device type. Every LP lives inside an initiative. Initiatives are created once per platform/partner/device combination and reused across campaigns.

**Data Source (DS)** — the numeric identifier for a traffic source inside Maven. Examples: DS12 = Facebook, DS18 = Taboola, DS14 = Bing. The DS determines URL structure, macro format, and alias domain usage.

**Landing Page (LP)** — a single Maven tracking record associated with one Instapage or Plasmic page. Contains the pre-lander URL (with tracking params), the client landing URL, the LP name, experience settings (EC, popup, continuation), and the Is Experiment flag.

**Landing Page Group (LPG)** — a container for one or more LPs. The LPG ID is what goes into the live ad campaign URL — it never changes while the campaign is running. LPGs enable A/B testing and traffic distribution between LP variants. Every LP must be inside an LPG for correct reporting.

**Media URL** — the final URL that analysts load into ad platforms (Taboola, Facebook, Bing, etc.). Generated by Maven from the LP + LPG + DS configuration.

**QA URL** — the Media URL with `&atnqa=1` appended. Used for testing the redirect chain without generating live tracking data.

## Current product stage

Maven Manage was recently refreshed (described internally as "just a refresh" — it did not resolve the core workflow pain points). The Build feature is being designed as the first meaningful workflow improvement on top of the refresh.

The primary user of the Build feature is **Sheba Lawrence**, Ad Ops Specialist. She is the only person who creates LPs and generates URLs in Maven today.

## What the Build feature must solve

Today, Sheba's LP creation workflow spans three systems: Asana (where the request ticket lives), Maven (where the LP and URL are created), and GTM (where UTM sources are configured). There is no connection between them.

The Build feature must:
1. Surface the Asana ticket data inside Maven so Sheba doesn't have to context-switch
2. Auto-resolve system decisions (DS, initiative, pre-lander on/off, alias domain, macros) from the traffic source selection
3. Guide Sheba through the four configuration sections in dependency order: Initiative → Traffic source → Landing page → Experience
4. Handle the UTM source async flow (LP/LPG save first, GTM setup, return to generate URL)
5. Write Maven ID and URLs back to the Asana ticket automatically after URL generation
